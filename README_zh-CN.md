# story-sandbox

[English](README.md) | [中文](README_zh-CN.md)

**自主故事生成沙盒。** 设定世界观和角色，然后看着他们自己发展。故事不可预测 — 连你都不知道接下来会发生什么。

输出到 Obsidian vault，自动生成关系图谱、时间线、Dataview 面板和 Canvas 画布。

---

## 工作原理

```
你设定：  世界观 + 角色 + 目标长度
              ↓
沙盒运行：角色自主行动（有限视角 = 涌现）
          世界自我修补（漏洞被解释，而非删除）
          新地点/角色自然出现
              ↓
输出：    Obsidian vault（场景、章节、关系图谱）
```

## 安装

```bash
pip install story-sandbox
```

然后添加到 Claude 配置：

```json
{
  "mcpServers": {
    "story-sandbox": {
      "command": "story-sandbox"
    }
  }
}
```

或使用 Claude Code CLI：

```bash
claude mcp add story-sandbox -- story-sandbox
```

## 快速开始

打开 Claude Code 说：

```
启动沙盒
```

Claude 会引导你完成：

1. **设定世界观**（4 个问题：时代、规则、冲突、基调）
2. **创建角色**（姓名、性别、性格、背景、动机、秘密）
3. **选择目标长度**（短篇 / 中篇 / 长篇）
4. **选择自主模式**（完全 / 半 / 约束）
5. **运行模拟**（角色自己行动）

## 工具

| 工具 | 说明 |
|------|------|
| `sandbox_init` | 初始化 vault 目录结构和世界文档 |
| `sandbox_add_character` | 添加角色（性格、背景、秘密） |
| `sandbox_get_state` | 读取当前状态（角色、伏笔、场景） |
| `sandbox_write_scene` | 写入场景文件并更新角色状态 |
| `sandbox_compile_chapter` | 编译场景为小说章节 |
| `sandbox_update_graph` | 更新 Obsidian Canvas 人物-地点关系图 |
| `sandbox_check_consistency` | 扫描命名冲突、时间线问题、关系漂移 |
| `sandbox_export` | 导出素材供 story-long-write 使用 |

## 核心机制

### 有限视角涌现

每个角色只知道该知道的事 — 不知道其他角色的内心、秘密或未目击的事件。这自然产生误解、误判和意外冲突。

### 世界自我修补

角色遇到未定义的地点或概念时，系统自动创建。发现不一致时，解释而非删除（"那段记忆被篡改了"）。

### 收敛曲线

| 阶段 | 进度 | 行为 |
|------|------|------|
| 自由生长 | 0-30% | 伏笔密度高，新角色/新地点频繁出现 |
| 回收期 | 30-70% | 伏笔回收优先级上升，不再新增 S 级伏笔 |
| 聚焦期 | 70-90% | 伏笔只减不增，角色聚焦核心冲突 |
| 终局 | 90-100% | 所有 S 级伏笔必须回收，最终对决 |

### 伏笔三级系统

| 等级 | 生命周期 | 用途 |
|------|----------|------|
| S（故事级） | 整本书 | 贯穿全书的核心悬念 |
| A（篇章级） | 10-20 轮 | 每个篇章解决的中等悬念 |
| B（场景级） | 3-5 轮 | 即时钩子 |

## Obsidian 集成

沙盒将所有内容直接输出到 Obsidian vault。在 Obsidian 中打开 vault，故事就活了 — 交互式可视化、关系图谱、时间线一目了然。

### 必装插件

| 插件 | 是否必须 | 用途 |
|------|----------|------|
| **Dataview** | 必须 | 驱动状态面板（角色状态、场景时间线、伏笔追踪） |
| **Canvas** | 自带 | 故事地图：角色、地点、关系连线 |
| **Graph View** | 自带 | 角色与地点的关系网络图 |

安装 Dataview：设置 → 第三方插件 → 浏览 → 搜索 "Dataview" → 安装 → 启用。

### Graph View 关系图谱

Graph View 只显示**角色和地点**（场景不参与图谱，避免过于拥挤）。颜色分组：

| 颜色 | 分类 | 示例 |
|------|------|------|
| 绿色 | 角色 | 林默、苏晴、陈维 |
| 橙色 | 地点 | 默记书店、档案馆、安全屋 |
| 紫色 | 世界规则 | 异能代价、核心规则 |

角色通过 wikilink 连接到他们访问的地点和其他角色。图谱随故事发展自动更新。

### Dataview 状态面板

`05-状态面板/面板.md` 包含自动刷新的 Dataview 查询：

- **角色状态表** — 每个角色的心情、原因、最后更新轮次
- **最近场景** — 最近 10 个场景的轮次、地点、情绪弧
- **世界文档** — 所有世界建设文档的类型和创建轮次

### 关系卡片（非矩阵表）

`04-关系图/关系索引.md` 使用每人一张卡的格式：

```markdown
## 林默

- **苏晴** — 75 信任。同盟搭档，一起调查腐败案。
- **陈维** — -30 敌对。确认是封锁身份的人。
- **方远** — 40 合作。在半盏茶馆秘密见面。

## 关系演变时间线

### 林默 <-> 苏晴：从陌生人到同盟
第1轮 陌生 → 第2轮 试探 → 第11轮 结盟 → 第15轮 并肩作战
```

### Canvas 故事地图

`06-画布/故事全景.canvas` 创建交互式故事地图：

- **角色节点**（上方）— 按阵营着色（蓝色=主角，红色=反派，黄色=灰色地带）
- **地点节点**（下方）— 所有重要地点
- **连线** — 标注关系值（"同盟 75"、"敌对 -30"）
- **故事概览节点** — 当前弧线和进度

在 Obsidian 中打开 canvas 文件，拖拽、缩放、探索故事结构。

### 场景文件格式

`02-场景/` 中每个场景文件包含：

```markdown
---
type: scene
round: 5
title: "深夜来电"
date: "2026-06-20"
location: "[[默记书店]]"
characters: ["[[林默]]", "[[苏晴]]"]
emotional_arc: "平静 -> 紧张 -> 释放"
tags: [scene, "轮次/5"]
---

# S005 - 深夜来电

## 场景正文
{200-800 字叙事片段}

## 结构化数据
### 关键事件
- 事件 1
- 事件 2
### 关系变化
- [[林默]] <-> [[苏晴]]：信任加深（亲密度 +10）
### 伏笔
- 新增：...
- 回收：...
```

所有 wikilink（`[[...]]`）自动在 Graph View 中创建连接。

## 示例

一个近未来都市的故事，人人有异能但使用会缩短寿命：

```
世界观：
  近未来都市，异能缩短寿命
  异能管理局 vs 异能解放组织
  阴郁、压抑

角色：
  林默（男）：前特工，书店老板，找失踪妹妹
  苏晴（女）：调查记者，异能：触碰读取记忆
  陈维（男）：企业家，妹妹失踪的幕后黑手
```

经过 50 轮自主发展后：

- 50 个场景，10 章小说
- 7 个角色（3 个初始 + 4 个涌现）
- 13 个地点，2 个城市
- 约 40,000 中文字
- 完整关系图谱和时间线
