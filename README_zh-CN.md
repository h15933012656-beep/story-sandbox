# story-sandbox

[English](README.md) | [中文](README_zh-CN.md)

**自主故事生成沙盒。** 设定世界观和角色，然后看着他们自己发展。故事不可预测 — 连你都不知道接下来会发生什么。

输出到 Obsidian vault，自动生成关系图谱、时间线、Dataview 面板和 Canvas 画布。

---

## 工作原理

```
你设定：  世界观 + 角色 + 目标长度
              ↓
沙盒运行：角色自主行动（有限视角 = 涌现）
          世界自我修补（漏洞被解释，而非删除）
          新地点/角色自然出现
              ↓
输出：    Obsidian vault（场景、章节、关系图谱）
```

## 安装

```bash
pip install story-sandbox
```

然后添加到 Claude 配置：

```json
{
  "mcpServers": {
    "story-sandbox": {
      "command": "story-sandbox"
    }
  }
}
```

或使用 Claude Code CLI：

```bash
claude mcp add story-sandbox -- story-sandbox
```

## 快速开始

打开 Claude Code 说：

```
启动沙盒
```

Claude 会引导你完成：

1. **设定世界观**（4 个问题：时代、规则、冲突、基调）
2. **创建角色**（姓名、性别、性格、背景、动机、秘密）
3. **选择目标长度**（短篇 / 中篇 / 长篇）
4. **选择自主模式**（完全 / 半 / 约束）
5. **运行模拟**（角色自己行动）

## 工具

| 工具 | 说明 |
|------|------|
| `sandbox_init` | 初始化 vault 目录结构和世界文档 |
| `sandbox_add_character` | 添加角色（性格、背景、秘密） |
| `sandbox_get_state` | 读取当前状态（角色、伏笔、场景） |
| `sandbox_write_scene` | 写入场景文件并更新角色状态 |
| `sandbox_compile_chapter` | 编译场景为小说章节 |
| `sandbox_update_graph` | 更新 Obsidian Canvas 人物-地点关系图 |
| `sandbox_check_consistency` | 扫描命名冲突、时间线问题、关系漂移 |
| `sandbox_export` | 导出素材供 story-long-write 使用 |

## 核心机制

### 有限视角涌现

每个角色只知道该知道的事 — 不知道其他角色的内心、秘密或未目击的事件。这自然产生误解、误判和意外冲突。

### 世界自我修补

角色遇到未定义的地点或概念时，系统自动创建。发现不一致时，解释而非删除（"那段记忆被篡改了"）。

### 收敛曲线

| 阶段 | 进度 | 行为 |
|------|------|------|
| 自由生长 | 0-30% | 伏笔密度高，新角色/新地点频繁出现 |
| 回收期 | 30-70% | 伏笔回收优先级上升，不再新增 S 级伏笔 |
| 聚焦期 | 70-90% | 伏笔只减不增，角色聚焦核心冲突 |
| 终局 | 90-100% | 所有 S 级伏笔必须回收，最终对决 |

### 伏笔三级系统

| 等级 | 生命周期 | 用途 |
|------|----------|------|
| S（故事级） | 整本书 | 贯穿全书的核心悬念 |
| A（篇章级） | 10-20 轮 | 每个篇章解决的中等悬念 |
| B（场景级） | 3-5 轮 | 即时钩子 |

## Obsidian 输出结构

```
your-vault/
├── 00-世界观/          世界文档（自动扩展）
├── 01-角色/            角色卡（每轮更新）
├── 02-场景/
│   ├── 01-第一章/      按章节分类
│   └── ...
├── 03-时间线/          时间线索引
├── 04-关系图/          关系卡片（每人一张）
├── 05-状态面板/        Dataview 面板
├── 06-画布/            Canvas 故事地图
├── 07-素材导出/
│   └── 小说正文/       章节正文
└── sandbox-state.json
```

### 图谱颜色

| 颜色 | 分类 |
|------|------|
| 绿色 | 角色 |
| 橙色 | 地点 |
| 紫色 | 世界规则 |

场景不参与图谱，避免过于拥挤。

## 示例

一个近未来都市的故事，人人有异能但使用会缩短寿命：

```
世界观：
  近未来都市，异能缩短寿命
  异能管理局 vs 异能解放组织
  阴郁、压抑

角色：
  林默（男）：前特工，书店老板，找失踪妹妹
  苏晴（女）：调查记者，异能：触碰读取记忆
  陈维（男）：企业家，妹妹失踪的幕后黑手
```

经过 50 轮自主发展后：

- 50 个场景，10 章小说
- 7 个角色（3 个初始 + 4 个涌现）
- 13 个地点，2 个城市
- 约 40,000 中文字
- 完整关系图谱和时间线
